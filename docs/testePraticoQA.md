Teste Prático QA Testing
📋 Sobre o teste
Este teste está estruturado em duas partes principais: UI Testing e API Testing. Você precisa implementar ambas as partes, e sua avaliação será baseada na qualidade da documentação, cobertura dos testes, capacidade de análise e pensamento crítico demonstrados.


Temos Nível 1 (Obrigatório) e Nível 2 (Diferencial) em cada um dos testes. Deixar de atender um ou mais requisitos do Nível 1 significa reprovação no teste. Atender um ou mais requisitos do Nível 2 contará como plus na avaliação.

🎯 O Desafio
O teste consiste em validar duas aplicações distintas:
- Plataforma de e-commerce: Sauce Demo
- API de sistema de reservas de hotel: Restful-Booker

📊 Níveis de Implementação

UI Testing (Sauce Demo)

Nível 1 (Obrigatório)
- Login com diferentes tipos de usuários
- Ordenação e filtragem de produtos
- Fluxo completo de compra
- Remoção de itens do carrinho
- Navegação entre páginas
- Logout

Nível 2 (Diferencial)
- Testes de responsividade
- Testes de acessibilidade
- Testes automatizados

API Testing (Restful-Booker)

Nível 1 (Obrigatório)
- Autenticação básica
- CRUD de reservas
- Validação de campos obrigatórios

Nível 2 (Diferencial)
- Testes de performance
- Testes de segurança
- Automação via scripts

🗄️ Estrutura da Documentação

UI Testing
- Plano de testes
- Casos de teste com resultados
- Análise de bugs
- Sugestões de melhorias
- Análise de riscos

API Testing
- Collection de requests
- Documentação dos cenários
- Resultados dos testes
- Análise de bugs
- Variáveis de ambiente

🛣️ Cenários Principais

UI Testing (Sauce Demo)
- Fluxo de autenticação
- Gerenciamento de produtos
- Processo de checkout
- Gestão do carrinho
- Navegação do site

API Testing (Restful-Booker)
- Fluxo de autenticação
- Gestão de reservas
- Filtros e buscas
- Tratamento de erros
- Validações de dados

🔧 Requisitos Técnicos Obrigatórios
- Documentação em Markdown
- Collection em JSON
- Evidências em capturas de tela por meio de imagem e/ou gravação
- README detalhado explicando como executar os testes, premissas assumidas, ferramentas utilizadas e outras informações relevantes

📝 Critérios de Avaliação
- Capacidade analítica e pensamento crítico
- Organização e clareza dos testes
- Cobertura do nível obrigatório
- Diferenciais acrescentados
- Sugestões de melhorias e identificação de riscos
- Qualidade das evidências

⏰ Considerações Finais
Caso não consiga completar o teste até o prazo definido:
- Garanta que tudo que foi construído esteja em funcionamento
- Relate na documentação quais foram as dificuldades encontradas
- Documente o que foi implementado e o que ficou pendente

📤 Envio da Solução
O projeto (documentação em Markdown e Collection em JSON) deverá ser hospedado em um repositório no seu GitHub. O link do repositório deverá ser fornecido por meio deste formulário. Não serão aceitos links de projetos enviados por outros meios.
